# Product Designer OS Blueprint / プロダクトデザイナー OS ブループリント

**ステータス / Status:** `Draft`

## ビジョン / Vision

プロダクトデザイナーが、判断の背景から実践の証拠までをつなげ、学びを再利用可能な資産に変えられる OS をつくります。

Create an operating system through which product designers can connect the rationale for decisions to evidence from practice and turn learning into reusable assets.

## ミッションと対象者 / Mission and target user

ミッションは、個人または小さなチームが、設計知識を見つけ、適用し、検証し、更新できるようにすることです。主な対象者は、プロダクトデザイナーと、その協働相手である PM、エンジニア、リサーチャーです。

The mission is to help an individual or small team find, apply, validate, and improve design knowledge. Primary users are product designers and their PM, engineering, and research collaborators.

## 中核能力領域 / Core capability domains

- Design System — foundations, tokens, components, patterns, governance, and decisions.
- Product Experience Patterns — product-level flows and recurring experience choices.
- UX Methodology and AI Product Design — methods and domain knowledge.
- Workflows, Prompts, Skills, and Agents — reusable ways to execute work.
- Case Studies and Learning Log — evidence, outcomes, and reflection.

## ライフサイクル / Lifecycle

`Knowledge → Workflow → Practice → Evidence → Reflection` を循環させます。知識はワークフローで実行可能になり、実践は証拠を生み、振り返りが知識と判断を更新します。

`Knowledge → Workflow → Practice → Evidence → Reflection` is a loop. Knowledge becomes executable through workflows; practice produces evidence; reflection updates knowledge and decisions.

## Prompt、Workflow、Skill、Agent / Prompt, Workflow, Skill, and Agent

- **Prompt** は、限定された一つの成果を得るための再利用可能な指示です。 / A reusable instruction for one bounded outcome.
- **Workflow** は、複数の知識や手順を結ぶ実行プロセスです。 / An execution process that connects multiple knowledge sources and steps.
- **Skill** は、テンプレート、参照、チェックを伴う反復可能な能力パッケージです。 / A repeatable capability package with templates, references, and checks.
- **Agent** は、目的に向けて Prompt と Skill を選び、Workflow を実行する主体です。 / An actor that selects prompts and skills and executes workflows toward an objective.

## プロジェクトリポジトリとの関係 / Relationship to project repositories

Product Designer OS は汎用の知識と実践の母艦です。個別プロジェクトのリポジトリは、製品固有の要件、実装、決定、成果物を保持します。両者は参照でつなぎますが、製品固有の事実をこの汎用リポジトリへ無批判に複製しません。

Product Designer OS is a home for general knowledge and practice. Individual project repositories retain product-specific requirements, implementation, decisions, and deliverables. Link the two, but do not copy product-specific facts into this general repository without a clear purpose.

## 成熟度 / Maturity

現在は **Core Architecture and Design System Foundation** 段階です。理想の将来像は、承認済みの設計資産、検証済みのワークフロー、実例に裏付けられたパターン、継続的な振り返りが相互に参照される状態です。

The current state is **Core Architecture and Design System Foundation**. The ideal future state has approved design assets, validated workflows, evidence-backed patterns, and continuous reflection that reference one another.
