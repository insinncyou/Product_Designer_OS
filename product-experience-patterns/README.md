# プロダクト体験パターン / Product Experience Patterns

**ステータス / Status:** `Draft`

## 目的 / Purpose

複数の接点、プロダクトの流れ、信頼や収益化など、繰り返し現れる体験上の課題に対する設計アプローチと注意点を蓄積します。

Capture design approaches and cautions for recurring experience problems across product flows, touchpoints, trust, and monetization.

## 今後追加する内容 / Planned content

- オンボーディングと初回体験 / Onboarding and first-run experience
- アクティベーションと協働 / Activation and collaboration
- 信頼、権限、設定 / Trust, permissions, and settings
- アップグレードと収益化 / Upgrade and monetization
- クロスチャネル体験とプロダクトレベルのワークフロー / Cross-channel experiences and product-level workflows

## メモ / Notes

インターフェースを複数コンポーネントで構成する再利用解決策は [Design System Patterns](../design-system/patterns/README.md) に置きます。ここでは、より広いプロダクト体験の意思決定を扱います。各パターンには、適用条件、代替案、リスク、実例を含めます。

Multi-component interface solutions belong in [Design System Patterns](../design-system/patterns/README.md). This area covers broader product-experience decisions. Each pattern will include its context, alternatives, risks, and examples.
