# デザインシステムパターン / Design System Patterns

**ステータス / Status:** `Draft`

ここでは、複数のコンポーネントを組み合わせた再利用可能なインターフェースとインタラクションの解決策を管理します。フォーム、データ管理、ワークフロー、フィードバックのカテゴリーは構造のみで、承認済みのパターンはまだありません。

This area manages reusable interface and interaction solutions composed from multiple components. The forms, data-management, workflows, and feedback categories are structural only; no approved patterns exist yet.

オンボーディング、収益化、権限、クロスチャネルなどの広い体験パターンは [Product Experience Patterns](../../product-experience-patterns/README.md) に置きます。

Place broader patterns such as onboarding, monetization, permissions, and cross-channel journeys in [Product Experience Patterns](../../product-experience-patterns/README.md).
