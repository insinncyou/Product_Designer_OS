# リリースプロセス / Release Process

**ステータス / Status:** `Draft`

リリース候補では、変更内容、影響を受ける資産、互換性、移行案、検証結果を記録します。承認されるまでは `Proposed`、利用可能になれば `Active`、置換後は `Deprecated` または `Archived` を使います。具体的なバージョニングとリリース頻度は `TBD` です。

For a release candidate, record changes, affected assets, compatibility, migration guidance, and validation results. Use `Proposed` until approved, `Active` when available, and `Deprecated` or `Archived` after replacement. Specific versioning and release cadence are `TBD`.
