# コントリビューションモデル / Contribution Model

**ステータス / Status:** `Draft`

変更は目的、根拠、影響範囲、代替案、検証方法を含む提案から始めます。token、component、pattern の変更は、関連資産と実装への影響を確認してからレビューします。承認前の案は `Draft` または `Proposed` とし、既存利用者を壊す変更は明示的な移行計画を添えます。

Changes begin with a proposal containing purpose, evidence, impact, alternatives, and validation. Review token, component, and pattern changes with their related assets and implementation impact. Keep unapproved proposals `Draft` or `Proposed`; attach an explicit migration plan to breaking changes.
