# テーマ戦略 / Theme Strategy

**ステータス / Status:** `Draft`

テーマは primitive を直接切り替えるのではなく、semantic token の値を文脈ごとに割り当てます。これにより component token と UI の参照を安定させます。ライト、ダーク、高コントラスト、ブランド差分の採用可否は、対象製品の要件と検証がそろうまで `TBD` です。

Themes assign contextual values to semantic tokens rather than switching primitives directly. This keeps component-token and UI references stable. Whether light, dark, high-contrast, or brand variants are adopted remains `TBD` until product requirements and validation are available.
