# レイアウト / Layout

**ステータス / Status:** `Draft`

レイアウトは情報の優先順位、読解、操作、レスポンシブな変化を支えます。将来の仕様では、コンテナ、グリッド、余白、ブレークポイント、密度、読み取り幅を、検証済みのニーズに基づいて定義します。

Layout supports information priority, reading, action, and responsive change. Future specifications will define containers, grids, spacing, breakpoints, density, and reading width from validated needs.
