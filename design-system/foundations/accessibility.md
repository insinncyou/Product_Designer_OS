# アクセシビリティ / Accessibility

**ステータス / Status:** `Draft`

アクセシビリティは後工程の監査ではなく、foundation、component、pattern、content の設計条件です。各仕様ではキーボード操作、フォーカス、意味構造、名前と説明、状態の伝達、コントラスト、動きの設定、エラーからの回復を確認します。

Accessibility is a design condition for foundations, components, patterns, and content—not a late audit. Each specification should consider keyboard operation, focus, semantic structure, names and descriptions, state communication, contrast, motion preferences, and error recovery.

具体的な適合主張は、対象製品と実装を検証するまで行いません。

Do not make conformance claims until the specific product and implementation have been evaluated.
