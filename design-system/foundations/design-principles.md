# デザイン原則 / Design Principles

**ステータス / Status:** `Draft`

この Draft は、詳細な製品方針が確定する前の共通判断基準です。

This Draft provides shared decision criteria before detailed product policy is established.

- **理解可能性 / Understandability:** ユーザーが次に何をできるかを理解できるようにします。 / Make next actions understandable.
- **一貫性 / Consistency:** 同じ意味と操作には一貫した表現を使います。 / Use consistent expressions for the same meaning and action.
- **アクセシビリティ / Accessibility:** 多様な能力、状況、支援技術で利用できることを基準にします。 / Design for diverse abilities, contexts, and assistive technologies.
- **回復可能性 / Recoverability:** エラーを予防し、起きた場合には安全な回復経路を示します。 / Prevent errors and provide safe recovery paths.
- **証拠に基づく改善 / Evidence-led improvement:** 仮説と決定を検証し、学びを記録します。 / Validate hypotheses and decisions, then record learning.
